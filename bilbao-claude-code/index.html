Speaker note: Todo desde PowerShell externo. Node.js es el único prerrequisito. winget viene preinstalado en Windows 11.

Speaker note: El atajo ccode posiciona en la carpeta del proyecto y lanza Claude en un solo comando. La extensión de VS Code da diffs visuales y enlaces clicables.

Speaker note: Los dos acceden al mismo modelo. La CLI es para el usuario técnico que quiere velocidad. La extensión es para quien prefiere ver los cambios antes de aprobarlos.

Speaker note: La línea de estado es el primer script útil. Muestra en tiempo real cuánto contexto queda. Es un ejemplo de cómo bash + Python extienden Claude Code.

Speaker note: Pasamos de la instalación al corazón del sistema: cómo Claude recuerda, cómo se especializa, y cómo se organiza un equipo de agentes.

Speaker note: Claude.md es probablemente el fichero más importante de todo el sistema. Es lo primero que deberían crear al empezar un proyecto.

Speaker note: El raíz gobierna todo. Los sub-CLAUDE.md añaden detalle local. Es como una constitución con leyes orgánicas por ministerio.

Speaker note: Ejemplo real: sin la regla de "textos litúrgicos exactos", Claude parafraseaba las lecturas. Con la regla, nunca lo hace. Una línea en CLAUDE.md previene un error sistemático.

Speaker note: Los skills son la forma en que Claude retiene conocimiento entre sesiones. Es el equivalente a tener un manual de procedimientos que el asistente lee antes de trabajar.

Speaker note: Los skills de Anthropic son genéricos y de proceso. Los del usuario son específicos del dominio. La combinación es lo que hace al sistema potente.

Speaker note: Este es el organigrama completo. Cada persona tiene su dominio. Los skills compartidos (library, notebooklm) los pueden invocar varias personas.

Speaker note: Estas son las 4 cosas que un usuario nuevo necesita saber el primer día. La más importante: carpeta como repositorio git, y CLAUDE.md dentro.

Speaker note: Pasamos a la parte más potente del sistema: el pipeline que convierte audio, PDFs y notas en conocimiento buscable.

Speaker note: Tres puertas de entrada. El sistema acepta cualquier formato y lo convierte en conocimiento estructurado. Whisper es gratuito vía API para transcripción.

Speaker note: Obsidian es un editor de notas gratuito con un sistema de enlaces bidireccionales. El vault es donde todo se almacena de forma permanente.

Speaker note: La clave es la búsqueda semántica. No necesitas recordar las palabras exactas — el sistema encuentra por significado. /research lanza 4 agentes en paralelo.

Speaker note: Este es el diagrama clave. La entrada se convierte en conocimiento permanente. La salida lo reutiliza en contextos nuevos. Es un ciclo que se refuerza.

Speaker note: Un buen sistema AI OS no delega todo al LLM. Python hace el trabajo pesado, determinista, sin errores de alucinación. Claude lo orquesta.
